### Anti-Patterns to Avoid
- **Inline styles for gradients:** Use Tailwind arbitrary values instead of `style={{}}` -- keeps everything in the utility class system, inspectable in devtools class list.
- **Separate mobile/desktop components:** Do NOT create `HeroSectionMobile.tsx` and `HeroSectionDesktop.tsx`. Use responsive classes within a single component -- prevents prop/logic duplication.
- **Using `<img>` instead of `next/image` for the home photo:** The home photo is the largest LCP element. `next/image` provides automatic optimization, srcset, and lazy loading. The logo can use either since it is small.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Font loading | `<link>` tags to Google Fonts CDN | `next/font/google` Playfair_Display | Zero layout shift, self-hosted, no CORS/privacy issues |
| Image optimization | Manual resize/compress pipeline | `next/image` with `fill` + `sizes` | Automatic WebP/AVIF, responsive srcsets, lazy loading |
| Vendor prefix for backdrop-filter | Manual `-webkit-` in CSS | Both `backdrop-blur-[16px]` AND explicit `-webkit-backdrop-filter` | Safari still requires webkit prefix for backdrop-filter |
| Logo resize | Manual image editing | Use `sips` CLI to resize PNG | macOS built-in, lossless, one command |

## Common Pitfalls

### Pitfall 1: Safari backdrop-filter Requires Webkit Prefix
**What goes wrong:** Frosted glass effect works in Chrome/Firefox but not Safari.
**Why it happens:** Safari requires `-webkit-backdrop-filter` in addition to `backdrop-filter`.
**How to avoid:** Always include both. Tailwind's `backdrop-blur-*` generates the standard property; add the webkit variant explicitly via arbitrary property: `[-webkit-backdrop-filter:blur(16px)]` or use an inline style for the webkit prefix.
**Warning signs:** Test in Safari during development.

### Pitfall 2: Gradient Arbitrary Value Syntax in Tailwind
**What goes wrong:** Tailwind arbitrary gradient values fail to compile when they contain spaces.
**Why it happens:** Tailwind parses spaces as class separators. In `bg-[linear-gradient(...)]`, spaces inside must be replaced with underscores.
**How to avoid:** Replace all spaces with `_` inside arbitrary values: `bg-[linear-gradient(180deg,rgba(44,44,44,0.1)_0%,...)]`
**Warning signs:** Classes appearing unstyled in the browser.

### Pitfall 3: next/image fill Mode Requires Positioned Parent
**What goes wrong:** `fill` image doesn't appear or overflows its container.
**Why it happens:** `next/image` with `fill` uses `position: absolute` -- needs a parent with `position: relative` and defined dimensions.
**How to avoid:** Parent must have `relative` class and explicit height (e.g., `h-[28rem]`). Already correct in current HeroSection code.
**Warning signs:** Image invisible or stretching beyond section bounds.

### Pitfall 4: Negative Equity Path Missing from Testing
**What goes wrong:** The negative equity layout (no stat panel) is never tested visually.
**Why it happens:** Demo/test data typically uses positive equity values.
**How to avoid:** Explicitly test with `earningsPerWeek={0}` and `earningsPerWeek={-100}`. Ensure the hero still looks premium without the stat panel.
**Warning signs:** Empty space where stat panel would be, or layout collapse.

### Pitfall 5: Logo PNG at Original Size (437KB)
**What goes wrong:** Hero loads a 437KB PNG file for a ~52px-tall logo.
**Why it happens:** Source logo is high-resolution print-quality PNG.
**How to avoid:** Resize to ~200px wide before adding to `public/`. Use `sips --resampleWidth 200 source.png --out public/logo-white.png`.
**Warning signs:** Lighthouse flags unnecessary large image download.

### Pitfall 6: Text Readability on Light Photos
**What goes wrong:** White text on frosted panels is readable, but the "LIVE AZ CO" mobile text fallback (no panel) blends into light sky areas of home photos.
**Why it happens:** Mobile logo has no frosted panel background, just floats over the gradient.
**How to avoid:** The gradient starts at only 0.05 opacity at top. Add a subtle text-shadow to the mobile logo text for guaranteed readability: `text-shadow: 0 1px 4px rgba(0,0,0,0.5)`.
**Warning signs:** Logo invisible on homes with light-colored exteriors or bright sky backgrounds.
