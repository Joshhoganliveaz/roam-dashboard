<feature>
  <name>Multi-horizon projection functions for three scenarios</name>
  <files>src/lib/types.ts, src/lib/defaults.ts, src/lib/calculations.ts, src/lib/calculations.test.ts</files>
  <behavior>
    Hold & Rent Projections (calcHoldAndRentProjections):
    - Returns array of projections at years [5, 10, 15]
    - Each projection includes: homeValue (compounded at appreciationRate), monthlyRent (compounded at rentGrowthRate 3%), monthlyMortgage (fixed or refi'd if marketRate < homeownerRate), monthlyCashFlow, annualCashFlow, rentMortgageGap, mortgageBalance, equity
    - homeValue at year 5 for $500K home at 4% = $500000 * 1.04^5 = $608,326 (approx)
    - rent at year 5 for $2000/mo at 3% = $2000 * 1.03^5 = $2,318/mo (approx)
    - When marketRate < homeownerRate, mortgage payment recalculated at marketRate (refi scenario)
    - When marketRate >= homeownerRate, keeps original mortgage payment
    - mortgageBalance uses calcRemainingBalance with ownershipMonths + year*12

    Move-Up Projections (calcMoveUpProjections):
    - Returns array at years [5, 10, 15]
    - Each projection: homeValue, mortgageBalance, equity, netProceeds (via calcNetProceeds), downPayment20Pct, purchasingPower (netProceeds/0.2), newMonthlyPayment (at marketRate from slider)
    - netProceeds at year 10 should subtract future mortgage balance, commissions, closing costs
    - purchasingPower = netProceeds / 0.2
    - newMonthlyPayment = calcMortgagePayment(purchasingPower * 0.8, marketRate, 30)

    Equity Play Projections (calcEquityPlayProjections):
    - Returns array at years [5, 10, 15]
    - Each projection: homeValue, mortgageBalance, equity, helocAvailable (80% LTV - mortgage balance), monthlyHelocPayment (interest-only at marketRate on helocAvailable), annualHelocCost
    - HELOC is interest-only: monthlyHelocPayment = helocAvailable * (marketRate / 12)
    - helocAvailable = max(0, homeValue * 0.8 - mortgageBalance)

    Edge cases:
    - Zero mortgage balance (cash purchase): mortgageBalance stays 0, full equity
    - Negative equity at horizon: helocAvailable clamped to 0
    - Zero interest rate: calcRemainingBalance handles this (simple division)
  </behavior>

    4. Implement calcHoldAndRentProjections using existing calcMortgagePayment, calcRemainingBalance
    5. Implement calcMoveUpProjections using existing calcNetProceeds, calcMortgagePayment
    6. Implement calcEquityPlayProjections using existing calcEquityPlay pattern (80% LTV), interest-only HELOC calc
    7. All functions use DEFAULTS.appreciationRate for forward projection (NOT historical annualized rate) per research pitfall #6
