    - validateCromfordData with 0 points returns { valid: false, reason: "..." }
    - generateShapedCurve(400000, 520000, 60, 0.054) returns array of ~60 ChartPoints with realistic variation (not a straight line), first point near purchasePrice, last point = currentValue
    - generateShapedCurve output has seasonal variation (values oscillate, not monotonic)
    - applyCurveConditionFactor on Cromford data points with factor 0.9 scales all pricePsf values by 0.9
    - applyCurveConditionFactor with factor 1.0 returns data unchanged
  </behavior>
  <action>
    1. Create `src/lib/cromford.ts`:
       - `validateCromfordData(data: CromfordDataPoint[]): { valid: boolean; pointCount: number; reason?: string }` -- checks for minimum 3 data points, valid date formats, positive pricePsf values
       - `generateShapedCurve(purchasePrice: number, currentValue: number, months: number, annualRate: number): ChartPoint[]` -- creates realistic market curve with seasonal variation (spring surge, winter dip) and longer-period market cycle noise. Uses deterministic seed from purchasePrice so same home always gets same curve shape. Pins last point to currentValue exactly.
       - `applyCurveConditionFactor(dataPoints: CromfordDataPoint[], factor: number): CromfordDataPoint[]` -- multiplies all pricePsf values by the factor, rounds to 2 decimal places
       - `getConditionFactor(condition: Condition, valueLow: number, valueHigh: number, homeSqft: number, currentPsf: number): number` -- derives the condition factor from actual comp range values. For "updated" returns 1.0, for "original" returns (valueLow / (midpointPsf * sqft)), for "remodeled" returns (valueHigh * 1.03 / (midpointPsf * sqft))

    2. Create `src/lib/cromford.test.ts`:
       - Test all behavior cases above
       - Test that generateShapedCurve output is not monotonically increasing (has realistic dips)

    3. Create `src/lib/claude-api.ts`:
       - `export async function extractCromfordData(imageBase64: string, mediaType: string): Promise<CromfordExtraction>` -- uses raw fetch to `https://api.anthropic.com/v1/messages` with Claude Vision
       - Uses the time-series extraction prompt from RESEARCH.md
       - Parses JSON response with try/catch, returns typed result
       - `CromfordExtraction` type: `{ city: string; chartType: string; dateRange: string; dataPoints: CromfordDataPoint[]; }`
       - If JSON parse fails, return a fallback object with `status: "fallback"` and empty dataPoints
